# PRD Validation Checklist

> Quality checklist for reviewing PRDs before approval.

## How to Use

1. Review each section of the PRD
2. Check off items that meet the criteria
3. Note any issues in the comments column
4. PRD is ready for approval when all required items are checked

---

## Problem Definition

### Required ✓

| Check | Criteria | Comments |
|-------|----------|----------|
| [ ] | Problem statement is clear and specific | |
| [ ] | Problem is backed by data/evidence | |
| [ ] | Impact is quantified (revenue, users, etc.) | |
| [ ] | Root causes are identified | |
| [ ] | Target users are defined | |

### Recommended

| Check | Criteria | Comments |
|-------|----------|----------|
| [ ] | User research supports the problem | |
| [ ] | Competitive context is provided | |
| [ ] | Problem aligns with strategy/OKRs | |
